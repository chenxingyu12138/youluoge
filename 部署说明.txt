游略阁 — 部署说明（含国内访问方案）
================================

GitHub 仓库：https://github.com/chenxingyu12138/youluoge


【为什么 youluoge.pages.dev 打不开？】
--------------------------------
网站已成功部署，海外可正常访问。
若你打不开，通常是本地 DNS 被劫持/污染（公司网络常见）。

自查：命令行执行 nslookup youluoge.pages.dev
  - 正常应解析到 Cloudflare IP（104.x / 172.x 段）
  - 若出现 221.x、127.0.0.1、::1 等，说明 DNS 被污染

临时解决：
  1. 把 DNS 改为 223.5.5.5 或 114.114.114.114
  2. 用手机 4G/5G 热点访问（绕过公司 WiFi）
  3. 使用下方 Gitee Pages 方案（国内最稳）


一、Gitee Pages 部署（★ 国内用户推荐）
--------------------------------

适合：国内访客、公司网络、无需 VPN

1. 注册 Gitee：https://gitee.com

2. 导入 GitHub 仓库
   右上角 + → 从 GitHub/GitLab 导入仓库
   → 填入 https://github.com/chenxingyu12138/youluoge
   → 导入（公开仓库）

3. 开启 Pages
   进入 youluoge 仓库 → 服务 → Gitee Pages
   → 部署分支：main
   → 部署目录：/（根目录）
   → 点击启动/更新

4. 获得访问地址（约 1 分钟生效）
   https://你的Gitee用户名.gitee.io/youluoge/
   例：https://chenxingyu12138.gitee.io/youluoge/

5. 开启访问统计（51.la，免费，国内可用）
   a) 打开 https://www.51.la 注册
   b) 添加网站，获得统计 ID（一串字符）
   c) 编辑仓库中 js/analytics-config.js
      把 LA_ID: '' 改为 LA_ID: '你的ID'
   d) 提交后 Gitee Pages → 更新（重新部署）
   e) 登录 51.la 后台查看 PV/UV/来源

注意：Gitee 为纯静态托管，/admin 自建统计不可用，请用 51.la。


二、Cloudflare Pages（海外 / 部分国内网络）
--------------------------------

正式地址：https://youluoge.pages.dev
（国内很多网络仍无法访问，见顶部说明）

部署：Workers & Pages → Pages → Connect Git → youluoge
Build command 留空，Build output: /

统计：KV 绑定 STATS_KV + 环境变量 ADMIN_PASSWORD
后台：https://youluoge.pages.dev/admin


三、Vercel（海外）
--------------------------------

https://youluoge.vercel.app（国内通常打不开）


四、常见问题
--------------------------------

Q: pages.dev 和 vercel.app 都打不开？
A: 正常，请用 Gitee Pages（第一节）。

Q: Gitee Pages 更新后内容没变？
A: 服务 → Gitee Pages → 点「更新」重新部署。

Q: 51.la 没数据？
A: 确认 analytics-config.js 已填 LA_ID 并重新部署 Pages。

Q: 想同时给国内外用户访问？
A: 国内分享 Gitee 链接，海外分享 youluoge.pages.dev。
