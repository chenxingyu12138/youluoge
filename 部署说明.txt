GitHub 仓库：https://github.com/chenxingyu12138/youluoge


【为什么 youluoge.pages.dev 打不开？】
--------------------------------
网站已成功部署，海外可正常访问。
若你打不开，通常是本地 DNS 被劫持/污染（公司网络常见）。

自查：命令行执行 nslookup youluoge.pages.dev
  - 正常应解析到 Cloudflare IP（104.x / 172.x 段）
  - 若出现 221.x、127.0.0.1、::1 等，说明 DNS 被污染

临时解决：
  1. 把 DNS 改为 223.5.5.5 或 114.114.114.114
  2. 用手机 4G/5G 热点访问（绕过公司 WiFi）
  3. 使用下方「国内可访问」方案（见第一节）

一、国内可访问部署（★ 推荐）
--------------------------------

方案 A — GitHub Pages（免费，先试这个）
  1. 打开 https://github.com/chenxingyu12138/youluoge/settings/pages
  2. Source 选「GitHub Actions」（仓库已含 .github/workflows/github-pages.yml）
  3. 推送 main 分支后 Actions 自动部署
  4. 访问：https://chenxingyu12138.github.io/youluoge/
  说明：部分公司网络仍可能慢或间歇打不开，但比 vercel/pages.dev 稳。

方案 B — 腾讯云 EdgeOne Pages（国内 CDN，免费档）
  1. 打开 https://pages.edgeone.ai/zh
  2. 注册 → 新建项目 → 上传站点根目录全部文件（含 index.html、css、js）
  3. 获得 *.edgeone.app 域名，国内访问通常较快

方案 C — 腾讯云 CloudBase 静态托管（有免费额度）
  1. 控制台 → 云开发 → 静态网站托管
  2. 上传 build 产物（本项目无需构建，直接上传根目录静态文件）
  文档：https://cloud.tencent.com/document/product/876

方案 D — 上码 Upma 等国内静态托管（拖拽上传即可）
  https://www.upma.cn （面向国内优化，有免费套餐）

访问统计（国内静态站通用）：
  a) https://www.51.la 注册 → 添加网站 → 获得 ID
  b) 编辑 js/analytics-config.js，LA_ID: '你的ID'
  c) 重新上传/部署站点
  注意：纯静态托管下 /admin 自建统计不可用，请用 51.la。


二、Cloudflare Pages（海外 / 部分国内网络）

四、常见问题
--------------------------------

Q: gitee.com/.../youluoge/pages 打不开？
A: 该 URL 本身不存在；且 Gitee Pages 服务已下线，请用第一节替代方案。

Q: pages.dev 和 vercel.app 都打不开？
A: 正常（DNS 污染），请用 GitHub Pages 或 EdgeOne/CloudBase。

Q: 51.la 没数据？
A: 确认 analytics-config.js 已填 LA_ID 并重新部署站点。

Q: 想同时给国内外用户访问？
A: 国内分享 GitHub Pages 或 EdgeOne 链接，海外分享 youluoge.pages.dev。
